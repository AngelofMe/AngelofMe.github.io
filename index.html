    如何设置Android调试桥（ADB）

    注意：在计算机上设置ADB只是方程式的一半，因为您还需要在智能手机或平板电脑上做一些事情才能接受ADB命令。

    手机设定

    1. 在手机上启动“设置”应用程序。
    2. 通常，在列表底部附近，点击关于电话选项。
    3. 然后，点按“内部编号”选项7次以启用“开发人员模式”。完成后，您将看到一条吐司消息。
    4. 现在回到“设置”主屏幕，您应该看到可以访问的新“开发人员选项”菜单。
    5. 进入那里并启用“ USB调试模式”选项。
    6. 您已完成电话设置过程的一部分。接下来，您需要在下面滚动并遵循针对特定操作系统的其余说明。

    在计算机上按照操作系统进行操作。

    如何在Microsoft Windows上设置ADB

    1. 下载适用于Windows的Android SDK平台工具ZIP文件。
    2. 将此ZIP文件的内容提取到易于访问的文件夹中（例如C：\ platform-tools）
    3. 打开Windows资源管理器并浏览到将这个ZIP文件的内容提取到的位置
    4. 然后从与此ADB二进制文件相同的目录中打开命令提示符。这可以通过按住Shift键并在文件夹中单击鼠标右键，然后单击“在此处打开命令窗口”选项来完成。（某些Windows 10用户可能会看到“ PowerShell”而不是“命令窗口”。）
    5. 使用USB电缆将智能手机或平板电脑连接到计算机。将USB模式更改为“文件传输（MTP）”模式。一些OEM可能会或可能不需要此功能，但最好还是将其保留在此模式下以实现一般兼容性。
    6. 在“命令提示符”窗口中，输入以下命令以启动ADB守护程序： adb devices
    7. 在手机的屏幕上，您应该看到一个提示，允许或拒绝USB调试访问。自然，您将希望在出现提示时授予USB调试访问权限（如果您再也不想看到该提示，请点按“始终允许”复选框）。
    8. 最后，从步骤6重新输入命令。如果一切成功，现在您应该在命令提示符（或PowerShell窗口）中看到设备的序列号。

    耶！现在，您可以在设备上运行任何ADB命令！现在，按照我们广泛的教程清单开始改装手机！

    如何在macOS上设置ADB

    1. 下载适用于macOS的Android SDK平台工具ZIP文件。
    2. 将ZIP提取到易于访问的位置（例如，桌面）。
    3. 打开终端。
    4. 要浏览到将ADB提取到的文件夹，请输入以下命令： cd /path/to/extracted/folder/
    5. 例如，在我的Mac上是这样的： cd /Users/Doug/Desktop/platform-tools/
    6. 使用兼容的USB电缆将设备连接到Mac。将USB连接模式更改为“文件传输（MTP）”模式。并非总是所有设备都需要这样做，但是最好只是将其保留在此模式下，这样就不会遇到任何问题。
    7. 终端位于与ADB工具相同的文件夹中后，您可以执行以下命令来启动ADB守护程序： ./adb devices
    8. 在设备上，您会看到“允许USB调试”提示。允许连接。
    9. 最后，从步骤＃7重新输入命令。如果一切成功，现在您应该在macOS的“终端”窗口中看到设备的序列号。

    恭喜你！现在，您可以在设备上运行任何ADB命令！现在，按照我们广泛的教程清单开始改装手机！

    尽管上面的指南肯定会起作用，但是一些经验丰富的macOS用户应该意识到，可以使用非官方的软件包管理器（例如Homebrew或MacPorts）在他们的Mac上安装ADB的简便方法。

    什么是PATH系统变量？

    Windows使用PATH来指定重要可执行文件的位置。通常，这些文件位于系统目录中，例如C：\ Windows和C：\ Windows \ system32。这就是为什么您可以在命令提示符下键入“ calc”以启动计算器，而不是“ chrome”以启动Google Chrome的原因。在安装应用程序（例如Java）时，有时会更改此变量。Java在安装时会将其自身添加到PATH变量中，这意味着您可以从任何目录使用Java。这对使用Java应用程序的人很有用，因此该程序不必尝试对Java位置进行硬编码。

    我们将修改PATH系统变量，以允许我们在Windows计算机上的任何地方使用adb或Fastboot。PATH在Linux上也存在，通常包含bin和sbin目录。我还将介绍如何将平台工具添加到Linux PATH变量。

Manager应用程序允许您调整根设置并安装模块，这些模块是可安装的模块。该应用程序还可以作为大量模块的存储库，这些模块可以轻松扩展设备的功能。使用Manager应用程序是充分利用此根方法的重要部分。

最后一步是验证一切是否正常。打开新安装的Manager应用程序。我们希望在应用程序中看到一堆绿色的复选标记。这表示您已成功获得root。做得好！

接下来，我们要确保您的设备仍然通过Google的SafetyNet检查。修改系统后，SafetyNet会阻止Google Pay之类的应用程序正常工作。Magisk通过使系统保持不变来解决此问题。选择“点击以开始SafetyNet检查”按钮。如果您在按钮下方看到两个绿色的复选标记，则表明您的设备已通过检查。Google偶尔会通过新的Android Security Patch更新来打破此错误，但是Magisk开发人员会在出现这种情况时发布修复程序。万一您在Custom ROM上，维护人员必须进行工作以解决SafetyNet出现的问题。您还必须注意，如果您修改设备的系统分区，SafetyNet可能会损坏。
